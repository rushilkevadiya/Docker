Docker is an open-source containerization platform used to build, ship, and run applications in isolated environments called containers.

It allows developers to package an application along with its libraries, dependencies, and configuration files, ensuring the application runs consistently across different systems.

2Ô∏è‚É£ Why Docker is Needed?

Before Docker, applications faced issues like:

* Dependency conflicts
* Different OS environments
* Difficult deployment
* ‚ÄúWorks on my machine‚Äù problem

Docker solves these problems by providing a standardized runtime environment, making applications portable and reliable.

3Ô∏è‚É£ What is Containerization?

Containerization is a lightweight virtualization method where applications run in isolated user spaces called containers while sharing the host operating system kernel.

Each container:

* Has its own filesystem
* Has its own processes
* Is isolated from other containers

4Ô∏è‚É£ Docker vs Virtual Machine

| Virtual Machine       | Docker Container      |
| --------------------- | --------------------- |
| Requires full OS      | Shares host OS        |
| Heavy and slow        | Lightweight and fast  |
| Uses more resources   | Uses fewer resources  |
| Startup takes minutes | Startup takes seconds |

5Ô∏è‚É£ Docker Architecture 

Docker follows a client-server architecture.

Main Components:

1. Docker Client
The interface used by users to interact with Docker.

2. Docker Daemon
The background service that builds, runs, and manages containers.

3. Docker Image
A read-only template used to create containers.

4. Docker Container
A running instance of an image.

5. Docker Registry
A storage location for Docker images (example: Docker Hub).

6Ô∏è‚É£ Docker Image 

A Docker Image is a blueprint for creating containers.

Characteristics:

* Read-only
* Contains application code and dependencies
* Built in layers
* Stored in Docker registries

7Ô∏è‚É£ Docker Container 

A Docker Container is a runtime instance of a Docker image.

Characteristics:

* Lightweight
* Isolated
* Can be started, stopped, or deleted
* Uses host OS kernel

8Ô∏è‚É£ Difference Between Image and Container

| Docker Image       | Docker Container |
| ------------------ | ---------------- |
| Blueprint          | Running instance |
| Static             | Dynamic          |
| Cannot be modified | Can be modified  |
| Stored in registry | Runs in memory   |

9Ô∏è‚É£ Dockerfile 

A Dockerfile is a text file containing instructions to build a Docker image.

Purpose:

* Automates image creation
* Ensures consistency
* Defines environment and dependencies
* Each instruction creates a layer, which improves build efficiency through caching.

üîü Docker Layers Concept

Docker images are built using layers:

* Each instruction adds a new layer
* Layers are cached
* Only changed layers are rebuilt
* This makes Docker fast and efficient.

By default, container data is temporary.

Docker Volumes are used to:

* Persist data
* Share data between containers
* Store data outside containers
* Volumes exist independently of container lifecycle.
